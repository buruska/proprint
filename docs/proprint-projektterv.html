Projektterv

Next.js alapú weboldal könyvkiadó számára

Ez a dokumentum egy első körös technikai és funkcionális terv ahhoz, hogy a kiadó publikus weboldala és admin felülete stabilan, bővíthetően és üzletileg használható módon elkészüljön.

Készült: 2026. 03. 31. Technológiai fókusz: Next.js + admin + emailes rendelés Cél: induláshoz szükséges feladatok és eszközök áttekintése

1. Projekt célja

A cél egy modern, gyors és könnyen kezelhető könyvkiadói weboldal elkészítése, ahol a látogatók megismerhetik a kiadót, böngészhetik a könyveket, kosárba tehetik őket, majd a rendelési igényüket emailben el lehet küldeni a kiadó munkatársainak.

A publikus felület mellett szükséges egy jelszóval védett admin rész, ahol a könyvek, az oldalszövegek és a kapcsolatfelvételi adatok is szerkeszthetők.

2. Fő funkciók

Publikus oldal
  • Bemutatkozó oldal a kiadóról
  • Könyvlista szűréssel vagy kategóriákkal
  • Egyedi könyv adatlap borítóval, leírással, árral
  • Kosár funkció
  • Rendelési adatok elküldése emailben
  • Kontakt oldal és kapcsolatfelvételi űrlap
Admin felület
  • Belépés jogosultságkezeléssel
  • Új könyv feltöltése
  • Meglévő könyv módosítása vagy archiválása
  • Oldaltartalmak szerkesztése
  • Kapcsolati adatok és beállítások módosítása
  • Beérkező rendelések naplózása opcionálisan

3. Ajánlott technológiai stack

Terület Ajánlott megoldás Miért jó választás
Frontend + backend Next.js App Router + TypeScript Egy projektben kezelhető a weboldal, az admin és a szerveroldali logika.
Stílusozás Tailwind CSS Gyors fejlesztés, konzisztens komponensalapú UI.
Adatbázis MongoDB Atlas + Mongoose Jól működik könyv- és tartalomjellegű adatokkal, gyors indulást ad és egyszerű admin CRUD-hoz kényelmes.
Hitelesítés Auth.js vagy egyedi admin auth Biztonságos admin belépéshez szükséges.
Email küldés Resend vagy SMTP alapú Nodemailer A rendelési kosár és a kontakt űrlap is megbízhatóan emailben továbbítható.
Képfeltöltés Cloudinary vagy S3 kompatibilis tárhely Könyvborítók és admin feltöltött képek biztonságos tárolására.
Hosztolás Vercel Next.js projekthez kényelmes deploy, preview és skálázás.

Ajánlás: első verzióhoz a MongoDB Atlas teljesen jó választás. Ha később nagyon összetett riportok, készletkezelés vagy pénzügyi integráció jönne, akkor lehet PostgreSQL-re váltani vagy eleve azt választani.

5. Rendelési folyamat javaslat

  1. A látogató könyveket tesz a kosárba.
  2. Megadja a nevét, email címét, telefonszámát és rendelési adatait.
  3. A rendszer összeállít egy strukturált rendelési emailt a kiadó részére.
  4. Opcionálisan automatikus visszaigazoló email is megy a vásárlónak.
  5. Az admin felületen a rendelés el is menthető, így utólag követhető.

Ez a megoldás egyszerűbb, mint egy teljes online fizetési rendszer, és jól illik olyan kiadókhoz, ahol a megrendelések kézi feldolgozása történik.

10. Javasolt első MVP

Érdemes első körben az alábbiakat elkészíteni: főoldal, rólunk oldal, könyvlista, könyv adatlap, kosár, emailes rendelés, kontakt oldal, admin bejelentkezés, könyvkezelés és alap tartalomszerkesztés.

A későbbi körökben jöhet blog, hírek, készletkezelés, többnyelvűség, kuponok, online fizetés vagy számlázási integráció.

11. Rövid összegzés

A projekt technikailag jól megvalósítható egyetlen Next.js alkalmazásban. Az első verzióhoz a legpraktikusabb összeállítás: Next.js + TypeScript + Tailwind + MongoDB Atlas + Auth.js + Resend/Nodemailer + Vercel.

Ez elég rugalmas lesz ahhoz, hogy a kiadó saját maga tudja kezelni a könyveket és a tartalmakat, miközben a rendelési folyamat egyszerű és üzletileg gyorsan bevezethető marad.

Következő lépésként ebből a tervből készíthető részletes adatbázis-séma, oldalstruktúra, majd egy induló Next.js projekt is.
